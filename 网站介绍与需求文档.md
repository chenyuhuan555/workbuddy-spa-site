# 猎头工作台（WorkBuddy）网站介绍与需求文档

**文档版本：** V1.0  
**适用地址：** `https://chenyuhuan555.github.io/workbuddy-spa-site/`  
**产品定位：** 面向猎头/招聘顾问团队的多人协作工作台，用于沉淀公司、岗位、候选人、推进过程、知识与会议纪要，并通过 AI 辅助提升寻访、筛选、沟通和交付效率。

---

## 1. 产品目标

### 1.1 要解决的问题

- 公司、岗位、候选人和推进信息分散在表格、聊天记录、招聘平台与个人电脑中，难以统一管理。
- 普通团队成员、业务编辑者和管理员需要不同的数据操作权限。
- 猎头在 JD 拆解、候选人筛选、沟通话术、客户报告、会议复盘等高频任务中，需要可复用的 AI 助手。
- 多设备协作时，业务数据不能仅保存在某一台电脑浏览器中。
- 旧页面已有业务数据，需要支持导入、迁移、导出与恢复。

### 1.2 成功标准

- 已授权用户在任意设备登录后可访问同一份团队业务数据。
- 管理员可管理成员；高级成员可编辑业务；普通成员只能浏览。
- 从公司创建岗位、从岗位维护 JD、从候选人建立推进记录可形成完整业务链路。
- 可导入/导出完整备份，降低迁移和误操作风险。
- AI 工具可基于岗位、候选人、公司或会议纪要上下文提供可执行建议。

## 2. 用户角色与权限

| 角色 | 登录 | 查看业务数据 | 编辑业务数据 | 创建公司/岗位/候选人/推进 | 管理成员账号 |
| --- | --- | --- | --- | --- | --- |
| 管理员 | 用户名 + 密码 | 是 | 是 | 是 | 是 |
| 高级成员 | 用户名 + 密码 | 是 | 是 | 是 | 否 |
| 普通成员 | 用户名 + 密码 | 是 | 否 | 否 | 否 |
| 未登录用户 | 不可访问 | 否 | 否 | 否 | 否 |

### 权限要求

- 未登录时不渲染业务数据。
- 普通成员不得看到可执行数据写入的入口，且前端与云端均需拒绝写入请求。
- 高级成员拥有与管理员相同的业务编辑权限，但不能创建、停用成员或重置成员密码。
- 管理员可创建普通成员/高级成员、停用或启用成员、重置成员密码。

## 3. 信息架构

网站主导航包含以下模块：

1. **工作台**：今日待办、关键业务数据、团队经营概览。
2. **公司**：客户公司档案、岗位、候选人推进和公司维度 AI 分析。
3. **候选人**：人才库、简历版本、画像、人才方向与岗位匹配。
4. **推进中心**：候选人与岗位的推进记录、漏斗阶段与进度。
5. **AI 工具箱**：寻访、筛选、沟通、交付、转换等 AI 工具。
6. **知识库**：团队沉淀、会议纪要、流程规范与公告。
7. **设置**：成员管理、数据备份与恢复、管理员级配置。

## 4. 核心业务功能需求

### 4.1 公司管理

#### 公司列表

- 展示公司名称、行业/城市、负责人、合作状态、开放岗位数、推进中人数、面试人数及最近更新时间。
- 支持按公司、行业、负责人、合作状态筛选。
- 空状态下提示迁移旧数据或创建公司。

#### 创建公司

- 管理员和高级成员可从“＋新建”或公司看板顶部创建公司。
- 创建字段：公司名称（必填）、行业、所在城市、负责人、合作状态。
- 合作状态包括：潜在客户、合作中、暂停合作。
- 创建成功后进入该公司详情，并写入本地安全快照与云端同步队列。

#### 公司详情

- 展示公司基础信息、开放岗位数和候选人推进数。
- 提供公司概览、AI 分析、岗位、候选人推进、动态与备注页签。
- 可从公司详情直接创建岗位，避免重复选择所属公司。

### 4.2 岗位管理

#### 创建岗位

- 管理员和高级成员在公司详情点击“新建岗位”。
- 岗位自动绑定当前公司。
- 创建字段：岗位名称（必填）、工作城市、薪资范围、负责人。
- 创建后进入该岗位详情并同步。

#### 岗位详情

- 展示公司、岗位名称、城市、薪资和负责人。
- 提供岗位信息、岗位候选人、人才推荐、岗位 AI 分析页签。
- “岗位职责与要求”应支持管理员和高级成员编辑多行 JD；内容可包含职责、硬性要求、优先条件、汇报线、薪资或客户偏好。
- 普通成员可阅读 JD，但没有编辑入口。

### 4.3 候选人管理

- 支持独立上传候选人简历，建立不依赖具体岗位的人才主档。
- 支持 PDF、Word、图片、文本等常见简历来源，并可粘贴简历文字。
- 可维护姓名、联系方式、当前公司、当前职位、城市、负责人、人才方向及简历文本。
- 支持多简历版本、原件/电子文本查看、信息提取、人才方向识别与去重提示。
- 支持从候选人详情查看岗位匹配结果，并将候选人加入岗位推进。

### 4.4 推进中心与漏斗管理

- 一条推进记录关联候选人、岗位和公司。
- 支持常见漏斗阶段：已发现、已触达、已回复、顾问预筛、待推荐、已推荐、客户接受、待面试、面试中、面试通过、Offer、接受 Offer、待入职、已入职、试用期、已转正、已终止。
- 每次阶段变更应记录时间、操作者、原因与备注，形成不可依赖界面顺序的推进历史。
- 支持按公司、岗位、负责人、阶段、匹配分筛选。
- 工作台需呈现关键漏斗指标、来源/上传人转化统计、阶段停留与待办提醒。

### 4.5 待办与团队经营

- 用户可创建、编辑、完成、删除个人待办，并关联候选人、公司、岗位或推进记录。
- 个人待办按账号隔离，不应被其他成员读取。
- 工作台展示业务概览、待办、人才池和业务进展，帮助顾问确定当天优先事项。

## 5. AI 工具箱需求

### 5.1 工具分类

AI 工具箱按以下业务场景提供能力：

- 寻访：JD 拆解、搜索策略、人才地图等。
- 筛选：候选人匹配、风险判断、画像补全等。
- 沟通：触达话术、跟进建议、面试准备等。
- 交付：候选人报告、客户推荐材料、项目复盘等。
- 转换：PDF 简历、BOSS 二维码、粘贴文本转电子简历。

### 5.2 PDF 与 BOSS 二维码转换

- 转换工具提供点击和拖拽上传区。
- **PDF 简历**：读取文字内容；必要时执行既有 OCR 兜底；提取成功后可生成并导出电子简历 TXT。
- **PNG 二维码截图**：识别二维码 URL，显示“打开 BOSS”链接；用户在 BOSS 页面登录并复制完整简历文字后，粘贴到工具中生成电子简历。
- PNG 二维码不应被当成简历正文直接生成电子简历。
- 对不支持的文件类型、二维码无法识别、PDF 无法读取等场景，应给出明确错误提示。

### 5.3 顾问助手

- 提供可展开的猎头顾问助手。
- 可回答通用猎头方法论、人才市场、搜索策略、薪资谈判与候选人沟通问题。
- 在公司、岗位、候选人或会议纪要上下文中提问时，应优先使用当前上下文。
- 会议纪要问答必须以该纪要文本为依据；内容未提及的事项应明确说明“会议纪要未提及”，不得编造会议事实。

## 6. 知识库与会议纪要

- 支持创建、编辑、查看、删除团队知识文档。
- 文档分类包括会议记录、工作沉淀、流程规范、团队公告及其他。
- 会议纪要按时间倒序展示，最新内容置顶。
- 用户查看某份会议纪要时，可在同一工作上下文中向顾问助手提问，不应因助手面板导致纪要正文不可读。
- 普通成员可浏览有权限的知识内容；编辑操作受角色权限控制。

---

## 7. 数据、同步与备份

### 7.1 主业务数据

- 主业务数据包括公司、岗位、候选人、推进记录、共享知识库、AI 分析结果及相关业务设置。
- 浏览器 IndexedDB 保存本地安全快照，保障弱网和刷新场景。
- 登录后通过 Supabase `workspace_state` 进行共享业务数据加载与云端推送。
- 云端版本冲突时不得静默覆盖他人更新，应提示用户刷新或处理冲突。
- AI API Key 属于本机敏感配置，不随共享业务数据同步。

### 7.2 备份与恢复

- 管理员可导出完整 JSON 备份。
- 管理员可导入完整 JSON 备份，用于从旧 `file://` 页面迁移历史数据或灾难恢复。
- GitHub Gist 仅作为独立管理员备份通道，不作为实时主数据库。

### 7.3 数据保护要求

- 数据写入前需验证当前角色。
- 云端数据访问使用 Supabase 身份认证与行级安全策略控制。
- 简历原件、二维码链接、联系方式等敏感业务信息仅向已登录、已授权用户展示。

## 8. 非功能性需求

### 易用性

- 页面应适配桌面端与常见平板宽度。
- 新建、保存、同步、上传、识别失败均应提供清晰中文反馈。
- 关键表单字段应标注必填项。

### 性能

- 大体积简历和电子简历文本采用分批/按需处理，避免阻塞页面。
- AI、PDF、OCR 等耗时任务需显示进行状态。

### 可靠性

- 保存成功需同时保留本地快照；网络恢复后由同步机制推送云端。
- 本地存储接近容量上限时提示导出备份。
- 用户应能通过导入备份恢复历史业务数据。

### 安全

- 前端仅使用 Supabase 可公开的 publishable key；不得暴露服务端密钥。
- 用户名密码登录态由 Supabase 管理。
- 不在网页或同步业务状态中保存第三方 AI 密钥明文。

## 9. 部署与运行边界

- 当前前端为静态网站，部署在 GitHub Pages。
- 静态网页可被任意设备访问，但业务数据需登录后才能查看。
- 后端能力由 Supabase 提供，包括认证、成员管理函数、业务状态同步与个人待办。
- GitHub Pages 仅负责静态资源发布，不能运行 Node.js 服务、SSR 或数据库。
- 已部署的静态页面地址：`https://chenyuhuan555.github.io/workbuddy-spa-site/`。
